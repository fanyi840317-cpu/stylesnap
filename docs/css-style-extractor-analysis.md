## 四、竞品弱点总结（我们的机会）

### CSS Scan Pro — 最大竞品但贵且有短板
1. **$69 太贵**：同类功能分散在免费工具中，用户认为不值
2. **无AI能力**：纯规则转换Tailwind，复杂CSS转换质量差
3. **无框架代码生成**：只能输出CSS，不能输出React/Vue组件
4. **无截图标注**：不能在截图上标注尺寸/颜色/字体
5. **云端收藏依赖其服务器**：离线无法用
6. **无设计系统导出**：只能单元素提取，不能一键导出整站设计令牌
7. **不提取视频/Lottie**：资产提取不够全

### CSS Peeper — 免费但功能受限
1. **Pro版付费墙严重**：关键功能锁在$29后面
2. **无实时编辑**：只能看不能改
3. **无Tailwind转换**：对现代前端不友好
4. **无组件提取**：只能看单个元素

### Tailscan — Tailwind专属
1. **只服务Tailwind用户**：不用Tailwind的人不会买
2. **$49也偏贵**：单一功能不值
3. **无颜色/字体/资产提取**

### DivMagic — 转换能力强
1. **$49/年订阅**：用户不喜欢订阅制
2. **无样式检查/编辑**：只有复制没有理解
3. **无设计工具**：无颜色/字体提取

## 五、创新突破点（5个方向）

### 创新1：AI驱动的智能转换 ⭐⭐⭐⭐⭐
**痛点**：CSS Scan的CSS→Tailwind是规则映射，遇到复杂CSS（嵌套选择器、CSS变量、calc()）就转换失败
**创新**：
- AI理解CSS语义，智能转换到Tailwind/React/Vue/HTML
- 截图→代码（不是提取，是从视觉理解生成）
- 自然语言描述修改："把按钮改大一点，颜色换成蓝色"→ AI修改CSS

**技术**：调用LLM API（Claude/GPT）做CSS理解和代码生成

### 创新2：设计令牌一键导出 ⭐⭐⭐⭐⭐
**痛点**：当前工具只能单元素提取，没有"一键提取整站设计系统"的能力
**创新**：
- 扫描全页 → 自动归类颜色/字体/间距/圆角/阴影 → 生成 Design Tokens JSON
- 导出格式：CSS变量 / Tailwind配置 / Figma Tokens / Style Dictionary
- 设计令牌Diff：对比两个网站的设计系统差异

**技术**：遍历DOM + computed style聚类 + 令牌标准化

### 创新3：截图标注+测量 ⭐⭐⭐⭐
**痛点**：设计师/PM需要"截个图，标注尺寸、颜色、字体"发给开发，目前需要两个工具
**创新**：
- 一键截图全页/区域
- 自动标注：元素尺寸、间距、颜色、字体
- 导出标注图片/PDF
- 标注链接可分享

**技术**：html2canvas + Canvas标注渲染

### 创新4：组件智能提取+代码生成 ⭐⭐⭐⭐
**痛点**：CSS Scan Pro能提取组件HTML+CSS，但不能转成可用代码
**创新**：
- 悬停组件 → 自动识别组件边界（不只是DOM子树）
- 生成：React组件 / Vue组件 / HTML+CSS / Tailwind组件
- 智能命名：AI根据视觉语义给class和component命名
- 去除冗余：自动清理不必要的wrapper div和样式

**技术**：DOM分析 + LLM代码生成 + heuristics

### 创新5：设计审计+可访问性检查 ⭐⭐⭐
**痛点**：开发者不知道自己的页面设计是否一致、是否可访问
**创新**：
- 一键审计：颜色对比度(ADA)、字体一致性、间距规律
- 设计一致性报告：哪些颜色/字号不在设计系统内
- 可访问性评分+修复建议

**技术**：WCAG对比度算法 + 统计分析

## 十、AI时代威胁重新评估（2026-05-26 补充）

### 关键发现：AI已经在读DOM了

之前的分析假设"AI只能从截图猜代码"——**这是错的**。

| 工具 | 输入方式 | 是否读DOM | 定价 |
|------|---------|-----------|------|
| **Pluck** | 点击组件→结构化prompt | ✅ 读computed style | Free 50次/月, $10/月无限 |
| **same.new** | 输入URL | ✅ headless浏览器+截图 | 免费/付费 |
| **CopyWeb** | URL或截图 | ✅ 服务端渲染+DOM分析 | 订阅制 |
| **ui.rip** | 输入URL | ✅ 真实浏览器引擎渲染DOM | 订阅制 |
| **Dupe** | Playwright+Claude Code | ✅ 完整DOM+computed style | 免费开源 |

### Pluck 是最直接的威胁

Pluck **和我们要做的几乎一样**：
- 点击组件 → 提取DOM结构 + computed style → 输出结构化prompt
- 支持导出到 Claude / Cursor / v0 / Bolt / Figma
- 免费50次/月，$10/月无限
- 日文评测：**DOM提取比截图方式精确3倍以上**
- 技术栈：Plasmo + Next.js + Hono + tRPC（和我们规划类似）

### 但是 Pluck 有短板

| 短板 | 说明 |
|------|------|
| **只输出prompt** | 不直接生成代码，还需粘贴到AI工具 |
| **无实时编辑** | 不能在页面上直接修改CSS预览 |
| **无设计令牌** | 不能一键提取全站设计系统 |
| **无截图标注** | 不能截图+标注尺寸 |
| **无Tailwind直接转换** | 靠AI间接转换，不是确定性映射 |
| **订阅制** | $10/月，用户不喜欢 |

### 重新评估威胁等级

| 原评估 | 修正后 |
|--------|--------|
| AI只能截图猜代码 ✕ | AI已经能读DOM精确值 ✓ |
| 精确性是我们的护城河 ✕ | Pluck也有精确性，护城河被削弱 |
| AI是3-5年后才威胁 ✕ | Pluck现在就在做，**立即威胁** |
| 截流CSS Scan可行 ⚠️ | 仍然可行，但要重新定位 |

### 新定位建议

如果"精确提取CSS"不再是独有优势，我们需要找**新护城河**：

| 方向 | 分析 |
|------|------|
| ❌ 精确值读取 | Pluck已有，非独有 |
| ⚠️ AI代码生成 | Pluck通过间接prompt实现，我们可以做**确定性直接生成** |
| ✅ **实时编辑+预览** | Pluck不做，这是交互层面的优势 |
| ✅ **设计令牌一键导出** | Pluck不做整站分析 |
| ✅ **截图标注** | Pluck不做 |
| ✅ **确定性Tailwind转换** | 规则映射比AI prompt更可靠 |
| ✅ **一次性买断$29** | Pluck $10/月订阅，长期看我们便宜 |

### 结论

**CSS样式提取器仍可做，但竞争格局比之前想的更严峻。** Pluck是直接的免费替代品。

核心策略调整：
1. **不能只做"提取"**——提取已被Pluck免费覆盖
2. **必须做"提取+编辑+导出"闭环**——Pluck只做提取，不做编辑和导出
3. **设计令牌导出是最大差异化**——全站设计系统提取，Pluck/CSS Scan都不做
4. **一次性买断是定价优势**——vs Pluck $120/年 = 我们$29终身
