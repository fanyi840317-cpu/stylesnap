## 一、产品定位

### 一句话描述

> **StyleSnap — 浏览器里的"设计提取→编辑→导出"闭环工具。从任何网站提取样式，实时编辑预览，一键导出为可用代码和设计令牌。**

### 核心价值主张

| 竞品 | 只能做 | 我们能做 |
|------|--------|---------|
| CSS Scan $69 | 提取 CSS | 提取 + 编辑 + 导出 |
| Pluck Free/$10月 | 提取 → 粘贴到AI | 提取 → 编辑预览 → 确定性导出 |
| Tailscan $49 | 只服务 Tailwind 用户 | 服务 CSS/Tailwind/React/Vue 全栈 |

**不是"又一个CSS检查器"，是"设计提取器+编辑器+导出器"三合一。**

### 目标用户

| 用户 | 核心痛点 | 现有解决方案 | 为什么不够 |
|------|---------|-------------|-----------|
| **前端开发者** | 看到好看的组件，想复制到自己的项目 | DevTools 手动抄 / CSS Scan 只出CSS | 需要的是 React/Vue 组件代码，不是原始CSS |
| **UI 设计师** | 想从竞品网站提取设计规范 | 手动截图+标注 / CSS Peeper 只看不能导 | 需要 Design Token JSON，不是一个个颜色值 |
| **PM / 产品经理** | 想截图标注尺寸颜色发给开发 | 截图+PS标注 / 两个工具切换 | 需要一键截图+自动标注，不想开两个工具 |
| **独立开发者** | 想快速复制网站风格搭建原型 | v0/Bolt从截图猜代码 | 猜的不准，需要精确值 + AI组合 |

### 2.2 六大竞争优势

#### 优势 1：闭环工作流（提取→编辑→导出）

| 步骤 | CSS Scan | Pluck | StyleSnap |
|------|----------|-------|-----------|
| 提取 | ✅ | ✅ | ✅ |
| 编辑预览 | ✅ 有限 | ❌ | ✅ 实时 |
| 导出代码 | ❌ 只有CSS | ⚠️ 需粘贴到AI | ✅ 直接生成 |
| 导出设计令牌 | ❌ | ❌ | ✅ |
| 截图标注 | ❌ | ❌ | ✅ |

**关键洞察**：所有竞品只做了"提取"这一步。提取之后怎么办？用户还要自己：
- 把CSS手动转成React组件（CSS Scan用户）
- 把prompt粘贴到AI工具再等10秒生成代码（Pluck用户）
- 用另外的工具截图标注尺寸（所有人）

我们做的是**提取后的一切**。

#### 优势 2：设计令牌一键导出（独有功能）

**这是最大差异化。没有任何竞品做这个。**

- 扫描全页DOM → 聚类颜色/字体/间距/圆角/阴影 → 生成标准化 Design Tokens
- 导出格式：
  - `tokens.json`（W3C Design Tokens 标准）
  - `tailwind.config.js`（直接可用）
  - `css-variables.css`（CSS 自定义属性）
  - Figma Tokens 格式

**为什么这是护城河？**
- 单元素提取是"点"能力，竞品都能做
- 整站设计系统提取是"面"能力，需要DOM全局遍历+统计算法，技术壁垒高
- AI工具（v0/Bolt/Pluck）只关注"复制一个组件"，没人做"提取一套设计规范"

#### 优势 3：确定性转换 vs AI猜测

| 转换场景 | Pluck的做法 | 我们的做法 |
|---------|------------|-----------|
| CSS → Tailwind | 提取CSS→生成prompt→粘贴到AI→AI猜Tailwind类名 | 规则映射+AI兜底，确定性优先 |
| CSS → React | 提取DOM→粘贴到Claude→等10秒→可能不对 | 提取DOM+样式→模板引擎直接生成 |
| 颜色值 | 原样输出 | 自动归类到设计令牌色板 |

**确定性转换 = 可靠 + 快速 + 离线可用**。AI只用于复杂场景兜底。

#### 优势 4：实时编辑预览

Pluck不做编辑。CSS Scan的编辑有限（只改值不能加属性）。

我们的编辑器：
- 修改CSS属性 → 页面即时更新
- 切换Tailwind类 → 实时预览
- 调整间距/颜色 → 拖拽滑块
- 编辑结果可直接导出

**这是交互层面的优势——AI工具永远做不到0.1秒即时反馈。**

#### 优势 5：截图+标注（设计师/PM刚需）

| 现有工作流 | 我们的工作流 |
|-----------|-------------|
| 1. 截图工具截图 | 1. 点击截图 |
| 2. 打开标注工具 | 2. 自动标注尺寸/间距/颜色/字体 |
| 3. 手动量尺寸标上去 | 3. 导出标注图 |
| 4. 导出发给开发 | 4. 或生成分享链接 |

**设计师和PM不需要懂CSS，只需要标注图。** 这个功能把用户群从开发者扩大到整个产品团队。

#### 优势 6：$29 一次性买断

| 竞品 | 价格 | 换算年成本 |
|------|------|-----------|
| CSS Scan Pro | $69 买断 | $69 |
| Pluck Unlimited | $10/月 | $120/年 |
| DivMagic | $49/年 | $49/年 |
| CSSPicker | $9.9/月 | $119/年 |
| **StyleSnap** | **$29 买断** | **$29** |

**vs Pluck：用3个月就回本。vs CSS Scan：便宜一半功能更多。**

### 4.2 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 扩展框架 | **Manifest V3** | Edge/Chrome 统一标准 |
| UI 框架 | **React 18 + Tailwind CSS** | 组件化开发，快速出UI |
| 构建工具 | **Vite + CRXJS** | 快速HMR，扩展开发体验好 |
| CSS 提取 | **getComputedStyle + getMatchedCSSRules** | 精确读取计算后样式 |
| Tailwind 转换 | **规则映射表 + AI兜底** | 300+常用映射覆盖90%场景 |
| React/Vue 生成 | **模板引擎** | 确定性输出，不依赖AI |
| 设计令牌 | **DOM遍历 + HSL聚类** | 统计算法归类颜色/间距 |
| 截图 | **html2canvas** | 成熟库，全页截图 |
| 标注渲染 | **Canvas 2D** | 自动标注尺寸/颜色 |
| AI 能力 | **Claude API / GPT API** | 仅复杂转换时调用 |
| 支付 | **Creem.io** | 中国个人开发者收款，3.9%+$0.4 |
| 存储 | **chrome.storage.local** | 纯本地，无服务端成本 |

### 4.3 核心数据流

```
用户悬停元素
    │
    ▼
Content Script: getComputedStyle(el)
    │
    ▼
过滤浏览器默认值 → 清洗后CSS
    │
    ├─→ 显示在Overlay浮层
    ├─→ 发送到Popup编辑器
    │       │
    │       ├─→ 实时编辑 → style注入 → 页面更新
    │       ├─→ Tailwind转换 → 规则映射/AI
    │       ├─→ React/Vue生成 → 模板引擎
    │       └─→ 复制到剪贴板
    │
    └─→ 设计令牌模式：
            DOM全页遍历 → HSL聚类 → 令牌标准化 → 多格式导出
```

### 4.4 设计令牌提取算法

```
1. 遍历全页可见元素
2. 收集 computed style: color, background, border, font, spacing, radius, shadow
3. 颜色聚类：HSL空间 → 色相±15°为一组 → 取中值为主色
4. 字体聚类：font-family去重 → 按使用频率排序
5. 间距聚类：收集所有margin/padding值 → 找8px网格规律
6. 圆角聚类：收集所有border-radius → 归类到4/8/12/16/24px
7. 输出：标准化令牌 + 频率/使用位置
```
